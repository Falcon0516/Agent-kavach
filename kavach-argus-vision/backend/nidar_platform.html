Section 01

The Problem

Why existing solutions are fundamentally broken.

The Crisis Is Not Lack of Apps

India has 30+ women's safety apps. Every Indian city has CCTV cameras. Every police station has an emergency number. Yet every 2 minutes, a crime against a woman is committed and the response remains catastrophically slow, fragmented, and reactive.

The real problem: Every existing tool sends one alert and waits for a human to coordinate the response. They're buttons, not agents. They inform, they don't act.

The Five Response Failures

Failure LayerWhat BreaksKavach Fix
AlertingOne-way notification, no coordination5 agents coordinate simultaneously
Police Response15–30 min average response timeAutomated 112 dispatch + CCTV activation
EvidenceNo automatic recording; evidence lostCloud evidence from second 1; CCTV preserved
TrackingNo cross-camera suspect trackingArgus mesh follows suspect across city
LegalVictims file FIR manually, days laterFIR pre-drafted automatically with evidence

What Exists — And Why It Fails

Nirbhaya App / Shakti / iCall

Sends one SOS alert. No autonomous follow-up. No CCTV. No evidence collection. No police coordination. A better button.

CCTV Networks (existing)

Passive recording only. Officers review footage days later — after DVRs have overwritten evidence. Zero real-time intelligence.

112 Emergency System

Requires manual call. No automatic context (location, threat level, attacker description). Caller must be able to speak calmly under duress.

Community Safety Apps (BeSafe etc.)

Passive heatmaps and community reporting. No emergency response. No real-time action. No coordination with law enforcement.

The gap Kavach fills: No system anywhere combines autonomous agent response + CCTV surveillance + volunteer network + police coordination + automatic evidence creation in one platform.

5 Autonomous AI Agents

When panic is triggered, all 5 agents fire in parallel — not sequentially. The orchestrator coordinates. No human required at any stage.

Agent 01 · Threat Assessor

Threat Intelligence Agent

🎯

Analyzes location coordinates against NCRB crime heatmap data, time of day, historical incident density, and nearby safe space availability. Classifies threat as LOW / MEDIUM / HIGH / CRITICAL and feeds risk level to all other agents to calibrate their responses.

Tools: NCRB API · Google Maps · Crime DB · Time analysis
Agent 02 · Responder Coordinator

Emergency Response Agent

🚨

Simultaneously initiates 112 police call with GPS + threat briefing, identifies nearest PCR van, alerts Kavach Sentinel volunteers in 500m radius, and notifies the user's trusted circle. Escalates automatically if no police confirmation in 90 seconds. Uses Shizuku + ADB for actual phone call initiation.

Tools: 112 API · Shizuku/ADB call · WhatsApp API · Volunteer network
Agent 03 · Family Alert

Trusted Circle Agent

👨‍👩‍👧

Sends WhatsApp message with live Google Maps location link to all trusted contacts. Provides real-time status updates every 60 seconds. If user marks "safe", sends confirmation. If no response within 5 minutes, escalates with second alert round. Manages the full communication loop with family until safety confirmed.

Tools: WhatsApp Business API · Firebase Realtime DB · Location stream
Agent 04 · Safe Route

Navigation & Guidance Agent

🧭

Identifies nearest registered Kavach safe space, open hospital, police station, or 24-hour shop. Delivers step-by-step voice navigation instructions via Android TTS — audible through earphones without user touching phone. Updates route dynamically if user moves off-path. Alerts if user appears stationary (potential incapacitation).

Tools: Google Maps API · Places API · Android TTS · Safe space registry
Agent 05 · Evidence Keeper

Legal Evidence Agent

⚖️

Immediately begins background audio recording uploaded in real-time to encrypted Firebase Storage — irrespective of what happens to the phone. Creates tamper-proof timestamp log. Pre-drafts IPC-appropriate FIR using Claude API with location, time, and any speech-detected incident description. Coordinates with Argus CCTV Agent to include camera footage in evidence package.

Tools: Firebase Storage · Claude API · IPC database · Section 65B formatter

ARGUS INTEGRATION

Section 05 · From Project Argus

The Argus Intelligence Layer

Project Argus is a decentralized AI CCTV investigation system — autonomous face/vehicle tracking across a camera mesh, court-admissible evidence generation, cross-jurisdiction relay. Kavach integrates Argus as its eyes on the street.

What Argus contributes: Argus proves that AI edge nodes on existing CCTV infrastructure can autonomously track suspects across a city in under 15 minutes. Kavach adds the trigger (panic button) and the response layer (agents) — Argus adds the eyes.

How Argus Works

Argus deploys AI Worker Nodes on existing CCTV cameras (Raspberry Pi + edge compute). Each node runs local face/vehicle recognition. All nodes connect via an encrypted P2P mesh (NATS channel). A Master Node at the police station orchestrates investigation sub-agents. One WhatsApp message starts a cross-city search in under 2 minutes.
